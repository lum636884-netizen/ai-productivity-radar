Fable 5 计划内访问今日到期，转 API-only $10/$50——用了就要重算账

是什么

Anthropic 把 Fable 5 的「计划内额度」（Pro/Max/Team/Enterprise 每周用量 50% 以内不额外计费）延长到 7/12 23:59 PT 后正式结束；7/13 起 Fable 5 仅按 API 计量收费，$10/百万输入、$50/百万输出。背景：Fable 5 于 6/9 上线，曾因出口管制全球暂停 19 天，6/30 解禁后压缩为六天窗口、封顶周用量 50%，本次到期即该窗口终点。

对你的价值

你此前把部分分类 / 轻任务路由给过 Fable 5（见 7/4「Fable 5 分类器误伤」）。今天起这些调用不再「含在套餐里」，而是按 $10/$50 计量——输出单价是 Opus 4.8（约 $25/百万输出）的 2 倍。对一人团队：若无 Fable 5 刚需，今天就把相关调用切走，别让它静默计费。

怎么应对

① 盘一遍你的脚本 / Skill 里是否显式或默认调用 Fable 5；② 轻量分类 / 抽取改用 Haiku 4.5 或 Sonnet（性价比更高）；③ 确需 Fable 5 能力的，改为显式 API 调用并设预算上限（可借本期扩展区 Copilot 的「会话开销封顶」思路，自建用量护栏）。

坑与验证

① $10/$50 为多源一致口径 + 官方 Fable 页，已证实；② 「计划内额度到期」只影响套餐内免费部分，不影响你主动的 API 按量使用；③ 时区坑——官方口径是 7/12 23:59 PT（= UTC+8 7/13 约 14:59），今天上午你可能仍在旧窗口尾巴。验证：在 Console 用量页看 Fable 5 是否开始计入 API 账单。

GitHub Copilot 加「会话开销上限」+ 浏览器工具 GA

是什么

Copilot CLI/SDK 现可给单次 agent 会话设 AI 额度上限，到顶自动收尾而非跑到底或等人手动停（公测，CLI ≥1.0.66 / SDK ≥1.0.5，覆盖模型调用、子代理与压缩等后台开销）；同时 VS Code 内浏览器工具转 GA、默认开启——agent 可导航页面、截图、验证 Web 应用。

对你的价值

竞品在补「无人值守 agent 花费封顶」这一环。你用 Claude Code 跑后台/自动化脚本，目前只有账户级用量限额、无逐会话美元封顶——这正是无人值守场景的成本盲区；浏览器工具 GA 与 Claude in Chrome（7/7 已推）同向，印证「agent 进浏览器」已是行业标配方向。

怎么应对

不必迁移。想给 Claude Code 自动化加类似护栏：脚本层记录每会话 token 用量并设阈值中断（API 响应的 usage 字段），或用 --max-turns 限步数兜底；同时留意 CC 是否跟进逐会话预算功能。

坑与验证

会话上限是公测功能，「到顶自动收尾」时任务的收尾质量未经充分验证；跨生态对比时注意两家口径不同（Copilot 按 AI credits、Claude 按 tokens）。验证：小任务设低上限试触发一次，看收尾行为。

建议行动

竞品参考——不迁移；本周给你的无人值守脚本自建一道预算护栏。

Claude Console 现可给 API Key 设过期时间（7/8 上线·补录）

是什么

Console 新建 API key / Admin key 时可选过期时间（预设 / 自定义 / 永不）；有效期 ≥7 天的 key 到期前 Anthropic 会邮件提醒创建者；Admin API 以 expires_at 字段可查每把 key 的到期时间。存量 key 不受影响。7/8 上线、此前未推，本条为补录。

对你的价值

你的雷达采集与营销自动化都在无人值守跑，脚本里的长命 key 一旦泄露就是长期风险敞口；给 key 设过期日是一道零成本安全护栏——到期自动失效，泄露损失有界。

怎么部署

Console → Settings → API keys → 新建时选过期时间；给「每日雷达」这类脚本建 90 天期的专用 key 并在日历记轮换日；存量长命 key 逐把替换为带过期的新 key。

坑与验证

过期只作用于新建 key，存量不会自动加期限——不替换就没有护栏；到期当天脚本会突然 401，收到提醒邮件务必及时轮换。验证：用 Admin API 列 key 核对 expires_at 是否生效。

建议行动

本周做——给自动化脚本换发带过期时间的专用 key，日历记轮换日。
