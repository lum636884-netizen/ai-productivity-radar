【终局】Fable 5 不收口也不四延：Max 永久内含（50% 限额），Pro 转 credits 并送 $100——一周拉锯落地

是什么

官方 @claudeai 在收口前宣布第三条路：7/20 起 Fable 5 纳入所有 Max 与 Team Premium 计划（周限额 50% 内使用，官方口径为永久安排、非临时促销）；Pro 与 Team Standard 用户改经 usage credits 使用（$10/百万输入、$50/百万输出），并获一次性 $100 credit。官方解释一如既往："Fable 的需求高且难预测"。这是 6/22 → 7/7 → 7/12 → 7/19 四个截止点拉锯的最终落地，也部分兑现了 07-17 报道的"回归订阅"承诺——按档位分层兑现。

对你的价值

按你的档位对号入座：若在 Max——Fable 5 永久保留在套餐内，过去一周准备的"切走"预案不必执行（保留为 credits 场景备用），高价值推理可长期用 Fable 5（50% 限额内）；若在 Pro——切换预案正式生效：轻任务留 Haiku/Sonnet，$100 赠送 credit 按 $10/$50 计价约可跑 200 万输入 + 160 万输出的混合负载，只花在最高价值的推理任务上。两档共同点：$10/$50 的计价没有变，Fable 5 依旧是"贵而强"的档位，分级路由结论不变。

怎么应对

① 今天确认你的订阅档位与应用内 Fable 5 标识（Max 应显示计划内、Pro 应出现 credits 余额与 $100 到账）；② Max：把 07-19 前"清仓"的高价值推理习惯常态化——50% 限额内放心用，超限部分再走分级路由；③ Pro：领取并核对 $100 credit，把它当"高价值推理专用金"，在 Console 设用量提醒防跑漏；④ 无论哪档：切换脚本保留（可逆原则），credits 价若未来调整可快速响应。

坑与验证

① "永久"是官方当前口径——按本案五周四改的历史，仍建议把它当"直到另行通知"；② $100 为一次性赠送，烧完即按量付费，别当长期额度；③ 7/17 起的 usage limit 异常事件（07-18 已跟踪）官方仍无单独说明，公告中"需求难预测"的配给逻辑或为背景，但无直接证据、不作归因。验证：应用内标识 + Console credits 余额 + 官方 X 原帖。

建议行动

今天核对档位与到账——Max：撤销切换、常态化使用；Pro：$100 只花刀刃、设用量提醒。分级路由框架两档通用，不变。

CC v2.1.215（7/19）：/verify 与 /code-review 不再自动运行——依赖自动校验的工作流要补显式调用

是什么

行为变更：Claude 不再自动触发 /verify（端到端验证改动）与 /code-review（代码审查）两个技能，需要时必须显式调用。此前这两个技能可能在提交前被模型自主拉起。

对你的价值

若你的开发/发布习惯默认"Claude 会自己校验"，这个假设从 215 起失效——无人值守流程里尤其危险：以前可能有的自动兜底没了。好处是行为更可预测、token 花销更省（不再有未请求的审查开销）。

怎么应对

① 在提交/发布类 Skill 或 hook 里显式写入 /verify 或 /code-review 步骤（把"自觉"改为"编码"——同我们 validate.py 门禁的打法）；② 复查你的自动化脚本是否隐含依赖自动校验行为。

坑与验证

行为变更类不会报错——只是"不再发生"，最容易静默漏掉；升级后第一次真实提交流程走一遍确认校验环节仍在。验证：changelog 原文 + 实测一次提交流。

建议行动

随升级补一行——把 /verify 写进你的发布流程模板，别依赖模型自觉。

是什么

OpenAI 推出 GPT-Live 实时语音模型：边听边说（每秒多次决策说话/停顿/打断/调工具）、九种重制语音、实时翻译；GPT-Live-1 为付费默认、mini 版为免费默认。与 GPT-5.6 家族全量 rollout 同周落地。

对你的价值

竞品观察为主，但一个远期场景值得记录：印尼语/中文/英语的实时翻译语音沟通——招商电话、加盟商答疑若有生产级实时翻译，语言门槛显著降低。当前你的语音场景不在 OpenAI 生态，不建议为此接入。

怎么应对

不动手。观察两点：① 实时翻译对印尼语的实际质量（等第三方实测）；② Anthropic 侧是否跟进同类实时语音能力。有印尼语实测数据后按 §9 评估。

坑与验证

发布日能力宣称与真实低延迟表现常有差距；小语种翻译质量尤其要等独立评测。验证：OpenAI 官方发布页 + 后续第三方语音实测。

建议行动
