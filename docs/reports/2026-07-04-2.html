1. API 限速全面提升：Sonnet/Haiku 现与 Opus 齐平，使用层级从 4 层并为 3 层✅ 已证实

是什么：2026-06-26，Anthropic 重构 API 使用限制体系：① 使用层级从 4 个精简为 3 个（Start / Build / Scale），有更高需求可联系销售申请 Custom；② Claude Sonnet 和 Haiku 的 RPM、ITPM、OTPM 限制全面对齐 Opus，同层级内三族模型限速相同；③ 所有模型（Haiku 3.5 除外）的缓存读取 token 不再计入 ITPM，只有未缓存的新输入 token 和缓存写入才计入。所有现有账号不降速，多数被自动升到更高层。
对你的价值：如果你的日常工作流（行研采集、文案批处理、竞对扫描）用的是 Sonnet 而不是 Opus，此前你的 RPM 上限可能显著低于 Opus——现在不再如此。具体影响：相同预算下，可以并发更多任务而不触发 429 错误；使用提示词缓存后，缓存读取部分不再消耗速率配额，多轮长文档对话的吞吐量大幅提升。
怎么部署：① 登录 platform.claude.com → Settings → Limits 确认当前层级和新限速数字；② 如果你的自动化脚本有手动限速逻辑（如 sleep 控制并发），可以相应调整；③ 启用提示词缓存的工作流现在可以放心扩规模，缓存命中不消耗 ITPM。
坑：层级是按月累计消费自动晋升，新账号仍从 Start 开始。若你之前的脚本有针对旧限速写死的延迟时间，实际限速已提升但脚本还在人为降速，记得检查。
🎯 行动建议：今天就查——登录控制台确认新层级和限速数字，同时检查自动化脚本里的 rate limit 配置是否仍贴合实际。

3. Claude Managed Agents 自托管沙箱 GA + MCP 内网隧道（研究预览）✅ 已证实MCP 隧道：研究预览

是什么：两个新能力打包上线：① 自托管沙箱（GA）——Managed Agents 的编排逻辑仍在 Anthropic 侧，但工具执行（bash、文件读写、代码运行）可以移到你自己的服务器或 Serverless 环境（支持 AWS Lambda、Vercel、Modal、E2B 等）；② MCP Tunnels（研究预览）——可以将 Claude 连接到你私有网络里的 MCP 服务器，无需把内部系统暴露到公网。
对你的价值：目前你是一人团队，多数工作流还在公共 API 层，这两个功能的直接刚需不高。但以下情形值得注意：① 如果未来你要处理加盟商的敏感协议数据或财务数据，自托管沙箱可以确保数据不离开你的服务器；② 如果你有内部 CRM 或私有数据库，MCP 隧道可以让 Claude 直接对接而无需数据外传。
怎么部署（先决条件）：自托管沙箱需要已使用 Claude Managed Agents（API / Claude Platform on AWS 均支持）；MCP Tunnels 目前是研究预览，需申请访问权限。两者均有详细 provider 指南（Vercel/Modal 入门门槛最低）。
🎯 行动建议：观望——现阶段对一人团队不是紧迫项。收藏留档，等业务规模需要数据合规隔离时再评估。MCP Tunnels 研究预览期间稳定性未知，不建议接入核心工作流。
