Claude Code v2.1.206/207（7/9-10）本周收官修复：auth/MCP/OAuth 稳定性 + 终端卡顿

是什么

两版以稳定性收尾，几条正中日常痛点：① 修复 OAuth 认证的 MCP 服务器在单次 token 刷新失败后就要求手动重新认证（这很可能正是"MCP 反复掉线要重授权"的根因之一）；② 修复 --mcp-config/.mcp.json 的 request_timeout_ms 被忽略、新会话里长任务 MCP 调用卡在 60 秒默认超时；③ 修复登录过期时所有模型报错误信息而非提示 /login；④ 修复 --resume/--continue 启动时键盘无响应；⑤ 修复流式输出超长列表/表格/代码块时终端卡顿、按键延迟；⑥ 云平台侧：Bedrock/Vertex/Foundry 开放 auto 模式、AWS/Vertex 默认切 Opus 4.8。

对你的价值

你整套雷达流水线跑在 Claude Code + MCP 上，①②③直接治你我这周反复遇到的"github MCP 掉线/需重授权/超时"类摩擦——升级后这类中断应显著减少。⑤对你写长报告/看长 diff 的体验是实打实改善。

怎么部署

升级 Claude Code 到 ≥ v2.1.207。若之前 MCP 频繁要重授权或长调用莫名超时，升级后观察是否消失；仍频繁的话再查 request_timeout_ms 配置。

坑与验证

OAuth 修复只解"单次刷新失败即掉线"这一类，不代表所有授权问题消失。验证：升级后连续跑几天，看 MCP 是否还像本周这样频繁需要重新授权。

建议行动

今天就升——尤其你被 MCP 掉线烦到，这两版正是冲着 auth/MCP 稳定性来的。
