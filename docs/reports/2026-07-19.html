CC v2.1.214（7/18）安全修复版：dir/** 允许规则会越界放行——升级并自查你的 settings

是什么

安全为主的一版：① 修复单段 dir/** 允许规则会自动放行整棵目录树里任何同名嵌套目录的写入（本应只匹配 <cwd>/dir）；② 修复多个 Bash 权限检查绕过（文件描述符重定向、>10,000 字符超长命令、zsh 下标语法、不安全的 help/man 自动放行）；③ docker 带守护进程重定向 flag（--url 等）现在会弹权限提示；④ 新增长任务进度心跳、OpenTelemetry 事件字段、修一批后台守护进程 / Windows 问题。

对你的价值

①② 直接关系无人值守安全：我们这类仓库的 settings 里就有 dir/** 形态的允许规则——旧版下它可能放行了比你以为更大的范围；权限绕过修复对"后台跑一晚"的信任边界是实打实收紧。这也延续了本周官方"防失控/收紧默认"的主线（212 会话上限 → 214 权限精确化）。

怎么部署

① 升级 ≥v2.1.214；② 自查一遍 .claude/settings*.json 里的允许规则：凡 Write(dir/**) / Edit(dir/**) 类单段通配，确认语义按"仅 cwd 下该目录"重新生效后行为符合预期（升级后首跑留意新增的权限提示，属正常收紧）；③ 用 docker 的注意新提示不是故障。

坑与验证

① 升级后部分此前"静默放行"的操作会开始弹提示——这是修复生效而非退化，别习惯性点掉，借机审一遍规则面；② Windows PowerShell 用户此版必升（权限检查绕过修复）。验证：升级后对一个嵌套同名目录做写入测试，确认不再被旧规则放行。
