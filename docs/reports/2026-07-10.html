Claude Code 权限安全：默认已改「手动批准」+ auto 模式的真实风险与正确用法

是什么

① v2.1.200（7/3）把默认权限模式从 Auto 改成 Manual（CLI/VS Code/JetBrains 全线，仅一行 changelog、无公告）——现在危险动作默认需你点头，auto 需显式开启。② v2.1.205（7/8）给 auto 模式加了"禁止篡改会话 transcript 文件"的安全阀。③ 背景警报：微软安全团队（6月）发现 Claude Code GitHub Action 可被 prompt 注入诱导泄露 CI/CD 密钥；4月有 GitHub issue 记录一次会话误删文件导致生产宕机+数据丢失。

对你的价值

你天天跑 CC agent（还在用后台子代理），这条直接关系到你的资产安全。默认改 Manual 是好事——别急着为图省事全开 auto。真实教训是：AI 层的安全防护是非确定性的（Simon Willison 明确不信任 AI 安全、只信 OS 级沙箱），一旦 agent 被诱导或误解指令，代价是密钥泄露或误删。

怎么部署

① 保持默认 Manual，只对**明确低风险、可回滚**的重复动作开 auto；② 关键项目用 OS 级隔离（受限文件访问 + 网络白名单 + 单独低权限账号/容器）兜底，别只靠 CC 的权限模式；③ 涉密操作（含 GitHub Action）警惕 prompt 注入——不要把密钥暴露在 agent 可读上下文；④ 破坏性命令（删除/部署/改配置）保留人工确认。

坑与验证

auto 模式"不是安全，只是比 dangerous 模式安全些"（Boris Cherny 原话）。验证你的隔离是否真生效：在沙箱里让 agent 尝试写 / 或访问白名单外域名，确认被 OS 层拦截，而非指望 agent 自觉。

建议行动

今天就查——确认你的 CC 是默认 Manual；给跑 agent 的关键仓库加一层 OS 级沙箱（限文件+限网络），别把安全全押在 auto 模式的 AI 判断上。

招商/Marketing：用 MCP 让 Claude 直连广告平台 / CRM 实时数据（区别于静态技能）

是什么

与 07-08 讲的"静态 Skills 技能链"是**不同层**：MCP 是"通用适配器"，让 Claude 直接调广告平台 / CRM 的 API 拉实时数据——不用再导 CSV，Claude 能直接读当前投放表现、实时预算、搜索词，相当于给它开了广告后台的只读权限。配合 Marketing Operations 类技能，可做线索生命周期建模与流程自动化。

对你的价值

招商 Marketing 的实时性痛点——"这周哪个渠道 CPL 涨了""预算烧到哪了"——以前要人肉导数据，用 MCP 后 Claude 直接连数据源答。对一人团队，等于把"数据搬运 + 初步诊断"这一层交给 agent，你只看结论。

怎么部署

① 从官方审核的 Connectors 目录（见 07-09）选广告平台 / CRM 的 MCP；② 先只给只读权限跑通"拉本周投放表现→给一句诊断"；③ 再叠加你的 product-marketing-context（07-08）让输出口径统一。

坑与验证

① 实时数据接入=把客户/投放数据暴露给 agent，务必最小权限 + 只读起步 + 看合规；② 结合精选①：给这类连数据的 agent 也上权限约束，别 auto 全开。验证：先对一个渠道跑只读诊断，人工核对数字与后台一致再扩。

建议行动

本周可试——挑一个你在用的广告平台/CRM，接只读 MCP，让 Claude 出一份"本周投放诊断"，核对准确度。

API 限速统一 + 用量分级简化为三档（Start / Build / Scale）

是什么

Claude 开发者平台提高 API 限速并简化用量分级：Sonnet 与 Haiku 的限速在各档均对齐 Opus；用量档位收敛为三档——Start / Build / Scale。

对你的价值

你若用 API 跑批量采集/摘要/招商话术，以前 Sonnet/Haiku 的低限速会在批量任务里卡你；现在对齐 Opus，同一档下批量吞吐更高、更少被限流。分档简化也让"我该在哪档、花多少"更清楚。

怎么部署

① 查你当前所在档（Start/Build/Scale）与对应 RPM/TPM；② 之前因 Sonnet/Haiku 限速被迫拆分的批量任务，可重新合并跑；③ 配合 Batch API（5 折）+ 缓存进一步压成本与限流压力。

坑与验证

限速提高 ≠ 无限，仍按档计；别把长期用量模型建在"促销/临时放宽"上。验证：跑一次你典型的批量任务，看是否还触发 429（限流），对比调整前后的吞吐。

建议行动

了解 + 按需调——确认自己所在档与新限速；把之前被限速拆碎的批量活合并重跑一次验证。
