【倒计时·剩 2 天】Opus 4.7 Fast Mode 7/24 移除——用了的话现在就迁，没用的确认一遍

是什么

平台已于 6 月底公告：7/24 起移除 Claude Opus 4.7 的 fast mode——届时对 claude-opus-4-7 带 speed: "fast" 的请求直接报错、不自动回退到标准模式。Opus 4.8 的 fast mode 不受影响。（本条为 §9 截止 ≤3 天再入选提醒，07-05 首报、07-11 曾预告。）

对你的价值

这是"硬错误"型截止：不迁移的后果不是变慢而是调用直接失败——无人值守脚本里尤其致命（半夜报错没人看见）。你的主力在 Opus 4.8 / Sonnet 5 上，大概率不中招，但值得花两分钟确认排除。

怎么应对

① grep -rn "opus-4-7" 扫一遍你的脚本 / Skill / 配置，凡带 speed: "fast" 的：改 claude-opus-4-8 + fast，或去掉 fast 参数；② 若无命中，本条到此为止；③ 7/24 后若见 4xx 报错提及 fast/speed，先想到这个截止。

坑与验证

易漏点：第三方工具 / 旧模板里硬编码的模型 ID（不止你自己写的脚本）；报错不回退是官方明确行为，别指望降级兜底。验证：7/24 后对旧 ID 发一个带 fast 的测试请求应得到明确错误，你的生产脚本应无此类调用。

建议行动

今天扫一遍——两分钟 grep 排除隐患，别把"硬错误"留给周五凌晨的无人值守任务。
