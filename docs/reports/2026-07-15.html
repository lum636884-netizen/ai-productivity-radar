Claude Code v2.1.208/209（7/14）大版本：终于修掉社区最恨的内存泄漏，长会话性能全面翻修

是什么

v2.1.208 是近几周最大的一版，直击社区高热痛点：① 修复多处长会话内存泄漏（粘贴图片、MCP stderr 累积至 64MB/服务器、LSP 文档常开、大工具结果——正是 GitHub 上"120GB RAM 被 OOM 杀掉"issue 群的病根），文件编辑缓存从 1000 个整文件收敛到 16MB、会话转录最多缩 79 倍；② 性能：高工具数场景工具调用最快 7×、权限规则匹配加缓存（多规则会话不再秒级卡顿）、200+ 行大表格不再卡死；③ 后台代理：给后台代理的回复投递失败不再丢失（保存并在重启后送达）、守护进程升级后 attach 失败已修、长上下文会话在 CLI 自动更新后被重置回 200k 已修；④ 安全：bypass 模式下含 $(…)/反引号的灾难性删除命令也会弹确认；⑤ 新增屏幕阅读器模式、vim 插入模式按键映射等。v2.1.209 随后回滚了一个误伤后台会话对话框的过宽守卫。

对你的价值

你的每日雷达就是"长会话 + 后台任务 + MCP"的典型负载——①③ 正是这套流水线的稳定性底座：跑一晚不再怕内存爬满、后台代理消息不再静默丢失；② 对多规则权限配置（我们仓库正是）的卡顿有直接改善；④ 给你的无人值守脚本多上一道防误删保险。

怎么部署

升级到 ≥ v2.1.209（209 含 208 全部内容 + 回滚修复）。若此前遇到过"越用越卡 / 内存爬升 / 后台代理没反应"，升级后跑一天长会话对比；无异常则无需其他动作。

坑与验证

① 内存修复覆盖已定位的泄漏源，不保证根除所有场景——GitHub 上 issue 仍开着的变体值得继续观察；② 208 的"后台会话对话框守卫"有误伤、209 才修——别停在 208。验证：升级后开长会话跑 2-3 小时，观察进程内存是否稳定在合理区间（此前社区实测 30 分钟即显著爬升）。

建议行动

今天就升到 v2.1.209——长会话稳定性与性能是这套雷达流水线的地基，这版全是实打实的修复。
