2. 踩坑警报：Sonnet 5 新 tokenizer 同文本多计 ~30% token🔶 可复现

怎么部署（验证）：用 API 的 token counting 接口对同一段典型工作文本分别在两个模型上计数，30 秒可自测；再看一次真实任务的账单对比。
坑：促销价 $2/$10 部分掩盖了这个问题，9 月 1 日恢复 $3/$15 后综合成本约为 4.6 的 1.3 倍。

4. 可靠性工程：分层降幻觉的工程化 Playbook🔶 可复现

是什么：多篇工程实践文章收敛出同一结论：没有单点技术能消除幻觉，可靠的是分层组合——检索事实垫底 → 允许模型"不确定时弃答" → 输出前验证（如 Chain-of-Verification：先答→自生成验证问题→独立回答→修正终稿）→ 约束输出格式。另一关键洞察：生产环境多数 agent 错误是上下文问题而非模型问题——正确的事实没有在正确的时刻出现在模型面前。
对你的价值：直接对应你"SOTA 模型仍交付错误结论"的痛点。你的行研和投资 thesis 验证工作流可以立刻套用 CoVe 模式；"弃答优于编造"应写进你所有 agent 的系统提示。
怎么部署：最小改动版：在现有工作流的产出环节后追加一步"生成 3-5 个验证问题并独立作答，不一致处修正或标注存疑"。
坑与验证：验证步骤会增加 30-60% token 消耗，对高价值交付（行研结论、给合伙人的材料）用，对日常草稿不用。
🎯 行动建议：今天就试——先在行研工作流加 CoVe 一步，对比修正率。
