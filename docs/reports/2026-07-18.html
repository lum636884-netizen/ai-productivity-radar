CC v2.1.212（7/17）：/fork 变后台会话 + 会话级搜索/子代理上限——官方把"防失控"护栏做进来了

是什么

① /fork 重设计：把当前对话复制成一个新后台会话（在 claude agents 可见），你继续在原会话干活；原来的会话内子代理行为改名 /subtask；② 会话级用量上限：WebSearch 每会话默认 200 次、子代理生成默认 200 个（可用环境变量调整）——官方明说是为了"止住失控的委派循环"；③ MCP 长调用自动转后台（默认 >2 分钟，阈值可配）；④ /resume 改为会话选择器（含已删除会话）并以后台会话恢复；⑤ 一批 worktree / hook / 计划模式误执行等修复。

对你的价值

这正是我们 07-13（Copilot 会话开销上限）与 07-16（自动修复分级放权）两期主线的官方回应：Claude Code 开始内建"每会话防失控"护栏。对你的无人值守流水线：子代理上限能挡住委派死循环烧配额；MCP 自动后台让长采集调用不再阻塞主线；/fork 后台化对"边跑雷达边开新任务"的并行习惯是直接改善。

怎么部署

升级到 ≥v2.1.212。两个默认值检查：若你的例行单会话搜索量可能超 200（本雷达约 6-10 次 / 天，安全），或并行子代理逻辑依赖大量派生，用 CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION / CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION 上调；习惯用 /fork 开分支讨论的，注意它现在落到后台会话列表里。

坑与验证

① /fork 语义变了——老肌肉记忆里"fork 出一个子代理在当前会话"现在要用 /subtask，别混；② 上限是新默认，触顶时行为是收尾而非硬断，但仍建议先在非关键任务上碰一次上限看表现。验证：升级后 claude agents 里能看到 /fork 出的后台会话即生效。

建议行动

本周末升——顺手确认两个 200 默认值对你的并行工作流是否够用；记住 /fork→后台、/subtask→会话内。
