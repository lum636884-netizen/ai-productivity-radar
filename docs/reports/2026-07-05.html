API 限速全面提升：Sonnet/Haiku 现与 Opus 齐平，使用层级从 4 层并为 3 层

是什么

2026-06-26，Anthropic 重构 API 使用限制体系：① 使用层级从 4 个精简为 3 个（Start / Build / Scale），有更高需求可联系销售申请 Custom；② Claude Sonnet 和 Haiku 的 RPM、ITPM、OTPM 限制全面对齐 Opus，同层级内三族模型限速相同；③ 所有模型（Haiku 3.5 除外）的缓存读取 token 不再计入 ITPM，只有未缓存的新输入 token 和缓存写入才计入。所有现有账号不降速，多数被自动升到更高层。

对你的价值

如果你的日常工作流（行研采集、文案批处理、竞对扫描）用的是 Sonnet 而不是 Opus，此前你的 RPM 上限可能显著低于 Opus——现在不再如此。具体影响：相同预算下，可以并发更多任务而不触发 429 错误；使用提示词缓存后，缓存读取部分不再消耗速率配额，多轮长文档对话的吞吐量大幅提升。

怎么部署

① 登录 platform.claude.com → Settings → Limits 确认当前层级和新限速数字；② 如果你的自动化脚本有手动限速逻辑（如 sleep 控制并发），可以相应调整；③ 启用提示词缓存的工作流现在可以放心扩规模，缓存命中不消耗 ITPM。

坑与验证

层级是按月累计消费自动晋升，新账号仍从 Start 开始。若你之前的脚本有针对旧限速写死的延迟时间，实际限速已提升但脚本还在人为降速，记得检查。

建议行动

今天就查——登录控制台确认新层级和限速数字，同时检查自动化脚本里的 rate limit 配置是否仍贴合实际。

Claude Managed Agents 自托管沙箱 GA + MCP 内网隧道（研究预览）

是什么

两个新能力打包上线：① 自托管沙箱（GA）——Managed Agents 的编排逻辑仍在 Anthropic 侧，但工具执行（bash、文件读写、代码运行）可以移到你自己的服务器或 Serverless 环境（支持 AWS Lambda、Vercel、Modal、E2B 等）；② MCP Tunnels（研究预览）——可以将 Claude 连接到你私有网络里的 MCP 服务器，无需把内部系统暴露到公网。

对你的价值

目前你是一人团队，多数工作流还在公共 API 层，这两个功能的直接刚需不高。但以下情形值得注意：① 如果未来你要处理加盟商的敏感协议数据或财务数据，自托管沙箱可以确保数据不离开你的服务器；② 如果你有内部 CRM 或私有数据库，MCP 隧道可以让 Claude 直接对接而无需数据外传。

怎么部署（先决条件）

自托管沙箱需要已使用 Claude Managed Agents（API / Claude Platform on AWS 均支持）；MCP Tunnels 目前是研究预览，需申请访问权限。两者均有详细 provider 指南（Vercel/Modal 入门门槛最低）。

Managed Agents 定时调度 + 凭证保险库公测：你的"每日例行 agent"可以官方托管了

是什么

Claude Managed Agents 两项能力进入公测：① Scheduled Deployments——给 agent 配一个 cron 表达式，到点自动启动新 session 完成任务，无需自建调度器，支持随时暂停/恢复/手动补跑；② Vault 凭证保险库——注册 API key 时绑定环境变量名和允许访问的域名，agent 沙箱里只有占位符，真实 key 在网络边界注入且只对白名单域名生效——agent 全程接触不到你的真实密钥。

对你的价值

这正是你当前"每日信息雷达"这类例行任务的官方标准解法。适用场景：每日竞对巡检、每周加盟线索汇总、定期行研摘要。Vault 机制解决了"要让 agent 调用第三方服务（如 CRM、邮件 API）但不敢把密钥交给它"的核心顾虑——对需要对接外部系统的营销自动化尤其重要。

怎么部署

在 Claude Platform 创建 Managed Agent → 配置 scheduled deployment（cron 表达式）→ 需要凭证的服务在 Vault 注册 key + 允许域名。官方文档有完整步骤。

坑与验证

公测阶段，SLA 未定；每次调度触发都是全新 session（无跨次记忆，需要持久状态得自己落盘到外部存储）；计费按实际 token 消耗，跑之前先小规模验证单次成本。

建议行动

本周内试——用一个低风险例行任务（如每日竞对页面摘要）验证调度稳定性和单次成本，跑通后再迁移核心工作流。

Claude for Small Business：Pro/Max 计划内免费开关，15 个开箱即用的小企业工作流

是什么

Anthropic 于 5 月 13 日推出的 Claude Cowork 内置开关（非新付费档位）：Pro/Max/Team 计划免费启用，含 15 个 agentic 工作流（覆盖财务、运营、销售、营销、HR、客服）+ 15 个可复用 Skill + 直连 QuickBooks、PayPal、HubSpot、Canva、DocuSign、Google Workspace、Microsoft 365 的连接器。典型工作流：发票催收、线索分诊（lead triage）、合同审查、营销活动创建、现金流监控、business pulse 报告。每个工作流执行前需你确认计划，熟悉后可放行端到端自动跑。注：5 月发布，非本周新闻——按"一人团队可部署实操"白名单纳入，此前未覆盖。

对你的价值

这基本是给你这类业务形态定制的能力包。最直接可用的三个：① 线索分诊——加盟咨询自动分类打分（与今日第 6 条 WhatsApp 机器人是同一漏斗的不同环节，这个在 Claude 内、零搭建成本）；② 发票催收——对加盟商的定期收款跟进；③ 营销活动创建——接 Canva 连接器直接产出物料。如果你已付费 Pro/Max，边际成本为零。

怎么部署

坑与验证
