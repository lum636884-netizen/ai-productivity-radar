1. Claude Code 子代理默认「后台运行」+ 完成通知，主线不再被阻塞✅ 已证实

是什么：7/1 的 Claude Code v2.1.198 起，用 Agent 工具派生的子代理默认在后台运行——主 Claude 不再干等，可继续手头工作，子代理完成或需要输入时通过 Notification hook 触发（新增 agent_needs_input / agent_completed 事件）。同版还修了「子代理/MCP 并行初始化」相关问题。
对你的价值：一人团队最缺的是并行度。以后可以让主线继续采集/写报告，同时甩一个子代理去跑「竞对批量摘要」或「招商线索初筛」这类独立活，完成时收到通知回来收口——把原本串行的两小时压成并行。配合你已有的 CLAUDE.md 铁律，子代理各自独立上下文、互不污染。
怎么部署：① 升级 Claude Code 到 ≥ v2.1.198；② 派生子代理时默认即后台，无需额外参数；③ 想收完成提醒，在 settings.json 配 Notification hook 监听 agent_completed / agent_needs_input（本仓库已有 hook 基建，加一条 matcher 即可）。
坑与验证：后台化意味着你不再"盯着"子代理——务必给子代理明确的完成判据和产物落点，否则它跑偏了你也不知道。验证：派一个"读 X 目录并返回摘要"的子代理，确认主线不阻塞、且 agent_completed 通知如期到达。

2. Claude in Chrome 正式 GA——浏览器内 AI 操作可进日常工作流✅ 已证实

是什么：7/1 起 Claude in Chrome 从预览转为正式可用（GA）。它让 Claude 直接在浏览器里读页面、填表单、跨标签页操作——相当于把 Computer Use 的浏览器子集做成了随手可用的扩展形态。
对你的价值：招商与 Marketing 有大量"人肉浏览器"活：查竞对官网/定价页、抓取行业目录里的潜在加盟商信息、在后台批量填表。GA 意味着不必再自己搭 Computer Use 环境，装上就能让 Claude 代你在浏览器里跑这些重复动作。
怎么部署：① 在 Chrome 装 Claude 扩展并用你的 Pro/Max 账号登录；② 从一个低风险只读任务起步（如"打开这 5 个竞对定价页，汇总成表"）；③ 涉及登录态/提交动作的，先在沙箱账号验证再上真实数据。
坑与验证：浏览器自动化天然踩「登录态、验证码、误点提交」三个雷——务必先只读、后写入，且对任何"提交/发送"动作保留人工确认。验证：先跑纯抓取任务，核对汇总结果与人工一致后，再逐步放开写操作。

3. 踩坑警报：Claude Code hook「静默失效」两个真 bug 已修（与本仓库 hook 直接相关）✅ 已证实

是什么：7/1 版本修了两个隐蔽的 hook bug：① 逗号分隔的 matcher（如 "Bash,PowerShell"）此前会静默从不触发；② SessionStart / Setup / SubagentStart hook 以退出码 2 失败时，stderr 被静默吞掉，现在会显示在 transcript 里。此外修复了 Windows 上 CLAUDE_ENV_FILE 与 SessionStart env 文件不生效的问题。
对你的价值：直接命中你——本仓库刚建了 .claude/hooks/session-start.sh（SessionStart hook，注入工作铁律）。这条警报的含义：(a) 若你写多事件 matcher，务必用官方支持的写法而非逗号拼接（旧版逗号会静默不触发）；(b) 若 hook 脚本以退出码 2 报错，旧版你根本看不到原因——现在升级后能看到 stderr，排障不再抓瞎。
怎么部署：① 升级 Claude Code；② 检查现有 settings.json 里的 hook matcher 是否用了逗号拼接，改为官方数组/多条目写法；③ 给 hook 脚本加 set -euo pipefail 并主动 echo 关键状态，配合修复后的 stderr 显示排障。
坑与验证：本仓库的 hook 目前是单事件（SessionStart）、无逗号 matcher，不受第一个 bug 影响；但要留意退出码——脚本末尾正常退出即 0。验证：故意让 hook 脚本 exit 2 跑一次，确认升级后能在 transcript 看到 stderr。
🎯 行动建议：今天顺手做——升级 Claude Code，并 grep 一遍自己 settings.json 里有没有逗号拼接的 matcher。

4. Sonnet 5 成为 Claude Code 默认模型 + 促销价 $2/$10（至 8/31）✅ 已证实

是什么：随 7/1 版本，Claude Sonnet 5 成为 Claude Code 的默认模型，原生 1M 上下文，且享促销价 $2/$10 每 MTok（输入/输出），有效期至 2026-08-31（此后恢复 $3/$15）。Sonnet 5 在编码/代理任务上接近 Opus 档，成本却远低于 Opus（$5/$25）。
对你的价值：你的日常采集、批量摘要、招商话术生成等大多数活不需要 Opus 顶配。默认切到 Sonnet 5 意味着同等产出下每月账单可观下降，且促销窗口内输入端再打 6-7 折。对一人团队的月度 API 支出是直接利好。
怎么部署：① 升级后默认即 Sonnet 5，无需操作；② 只在确实需要最强推理的硬任务（复杂重构、深度行研）手动切 Opus 4.8；③ 促销窗口内把可批处理的活尽量前移，叠加 Batch 5 折进一步降本。
坑与验证：① Sonnet 5 换了 tokenizer，同样文本 token 数比旧 Sonnet 约 +30%——按 token 计的预算/上限要重新标定；② 促销价 8/31 到期，别把长期成本模型建在促销价上。验证：跑一批代表性任务对比 Sonnet 5 与 Opus 的产出质量与 usage，按路由分级用模型。
🎯 行动建议：本周做——把日常低风险任务留在 Sonnet 5 默认，仅对硬任务手动升 Opus；用 count_tokens 重新标定预算上限。
